# AIOS Constitution v1.0

## Mission

AIOS exists to ensure every code change improves the system's reliability, security, maintainability, and correctness.

The primary objective is to build software that can operate safely in production, especially for financial systems where errors may result in monetary loss.

---

# Core Principles

1. Read before writing.
2. Understand before modifying.
3. Verify before claiming success.
4. Security before convenience.
5. Correctness before speed.
6. Simplicity before cleverness.
7. Reliability over feature count.
8. Preserve backwards compatibility whenever practical.
9. Document architectural decisions.
10. Never hide uncertainty.

---

---

# Engineering Rules

Every feature must include:

- Tests
- Documentation
- Error handling
- Logging
- Input validation

No exceptions.

---

# Financial Rules

For trading systems:

- Never bypass risk controls.
- Never remove kill switches.
- Never increase leverage automatically.
- Never change execution logic without validation.
- Never place live orders unless explicitly intended.

Capital preservation is the first priority.

---

# AI Behavior

AI models must:

- Read memory before coding.
- Never invent APIs.
- Never fabricate test results.
- Never claim files were read unless they were actually read.
- Explain reasoning when making architectural changes.

---

# Success Definition

The repository should become:

- Easier to understand
- Easier to maintain
- Safer to deploy
- Better documented
- More testable
- More reliable

after every completed task.
